## 一、项目能做什么

这是一个可在本机或局域网内运行的电商 AI 出图网站，主要功能包括：

- 上传商品图并生成电商图片
- 支持场景图、白底图、模特图、海报图、细节图、痛点图
- 支持多张图片批量生成
- 当前仅生成纯图片结果
- 支持历史记录、再次生成、下载结果
- 支持 Google 官方 Gemini 和 Gemini 兼容中转站

## 五、如何填写 API

进入网站后，先打开“设置”页面。

你会看到以下字段。

### 1. 默认 API Key

这是最重要的一项。

#### 如果你使用 Google 官方 Gemini

- 这里填 Google AI Studio 生成的官方 `Gemini API Key`
- 官方入口：[Google AI Studio](https://aistudio.google.com/app/apikey)

#### 如果你使用 Gemini 兼容中转站

- 这里填中转站提供的 Key
- 不是填浏览器 Cookie
- 不是填账号密码
- 不是填网页地址

### 2. Gemini Base URL / 中转站地址

#### 如果你使用 Google 官方 Gemini

- 这一项留空

#### 如果你使用中转站

- 填中转站文档里给你的 `Base URL`
- 必须是接口地址，不是官网首页，不是文档页地址

示例（只是格式示例，不是你的真实地址）：

```text
https://example.com/v1beta
https://api.example.com/google
```

### 3. API 版本

建议默认填：

```text
v1beta
```

如果中转站文档明确要求别的版本，再按文档填写。

### 4. 自定义请求头 JSON（可选）

只有在中转站文档明确要求“额外请求头”时才填写。

例如：

```json
{"Authorization":"Bearer 你的Key"}
```

如果中转站不要求，这一项留空即可。

### 5. 默认文本模型

建议填写：

```text
gemini-2.5-flash
```

### 6. 默认图像模型

建议填写：

```text
gemini-2.5-flash-image
```

### 7. 素材存储目录

建议先保持默认，不要改：

```text
C:\Users\AEboli\Documents\Codex\data\assets
```

### 8. 并发任务数

建议先填：

```text
1 或 2
```

原因：

- 更稳定
- 更容易排查问题
- 不容易一下子把额度打满

### 9. 测试连接

设置页里先点“测试连接”，看到成功后再点“保存设置”。

### 10. 临时 API Key

创作台里如果你填写了“临时 API Key”，它会优先覆盖默认 API Key。

如果你想一直使用设置页里保存的默认 Key，就把临时 API Key 留空。

## 六、如何开始生成图片

进入“创作台”，按照下面步骤操作。

### 1. 上传商品图

- 支持上传多张图片
- 第一次建议先上传 1 张测试

### 2. 填写商品信息

常见字段包括：

- 商品名
- SKU（可选）
- 品牌名（可选）
- 品类
- 核心卖点
- 限制词 / 禁止元素
- 补充说明

### 3. 选择目标市场参数

你可以选择：

- 国家
- 输出语言
- 平台
- 品类

### 4. 选择图片生成类型

可选类型包括：

- 场景图
- 白底图
- 模特图
- 海报图
- 细节图
- 痛点图

可以多选。多选后系统会分别生成对应版本。

### 5. 选择规格

你可以设置：

- 比例：`1:1`、`4:5`、`3:4`、`16:9`、`9:16`
- 分辨率：如 `1K`
- 数量：每种组合生成几张

### 6. 生成结果类型

当前系统只生成纯图片结果。

### 7. 提交任务

点击“提交任务”后，系统会按这个顺序处理：

1. 生成营销文案
2. 生成优化提示词
3. 调用图像接口出图
4. 写入数据库和本地素材目录

---

## 七、第一次测试建议这样填

这是最稳的一组测试参数：

- 上传 1 张商品图
- 图片类型只勾选：`场景图`
- 比例：`1:1`
- 分辨率：`1K`
- 数量：`1`
- 先不做太多组合

这样最容易判断问题是：

- API 设置问题
- 图片生成问题
- 参数设置太多导致的问题

## 八、如何查看结果、下载和再次生成

### 1. 任务详情页

提交任务后，可以在任务详情页看到：

- 原图
- 纯图片
- 生成状态
- 优化提示词
- 文案内容

### 2. 下载图片

每张结果图下方都有“下载”按钮。

### 3. 历史记录

在“历史”页面可以查看以前生成过的任务，并支持筛选：

- 商品名
- SKU
- 平台
- 国家
- 语言
- 分辨率
- 时间

### 4. 再次生成

在任务详情页可点击“再次生成”，复制旧任务参数重新跑一次。

## 十、如何让局域网其他电脑访问

### 1. 启动网站

推荐双击：

- `启动正式版.bat`

或手动运行：

```bat
cd /d C:\Users\AEboli\Documents\Codex
npm start -- --hostname 0.0.0.0 --port 3000
```

### 2. 查看本机局域网地址

双击：

- `查看局域网地址.bat`

或手动运行：

```bat
ipconfig
```

找到当前网卡的 IPv4 地址，例如：

```text
192.168.1.23
```

### 3. 其他电脑访问

在其他同一局域网设备上打开：

```text
http://192.168.1.23:3000
```

### 4. 访问失败时检查

- 两台电脑是否在同一局域网
- 防火墙是否拦截 Node.js
- 路由器是否禁止设备互访
- 网站是否确实以 `0.0.0.0` 启动

## 十一、常见问题排查

### 1. 打不开网站

常见原因：

- 没在正确目录运行
- 启动窗口被关闭
- 端口被占用

换端口示例：

```bat
npm start -- --hostname 127.0.0.1 --port 3001
```

### 2. 页面能打开，但生成失败

常见原因：

- API Key 无效
- 中转站 Base URL 填错
- 自定义请求头 JSON 格式错误
- 模型名填错
- 一次生成组合太多

建议先用最小配置测试：

- 1 张图
- 1 种图片类型
- 1 个比例
- 1 个分辨率
- 数量 1

### 3. 图片生成成功，但显示不出来或下载失败

这个问题之前已经修复。

如果你仍然遇到：

1. 关闭网站
2. 重新启动 `启动正式版.bat`
3. 浏览器强制刷新：`Ctrl + F5`

### 4. 端口被占用

如果 `3000` 被占用，可改端口：

```bat
npm start -- --hostname 127.0.0.1 --port 3001
```

### 5. PowerShell 执行策略报错

尽量直接双击 `.bat` 文件，或者改用 `cmd`。
