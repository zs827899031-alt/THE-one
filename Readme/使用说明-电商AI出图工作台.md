# 使用说明-电商AI出图工作台

对应对象：基于 Gemini 的电商生图工作台

## 1. 文档用途

本文档面向最终使用者、项目接收方和日常运营人员，说明如何安装、启动、配置和使用当前系统，不包含内部开发、打包或安装器制作细节。

## 2. 产品简介

本项目是一个面向电商团队的本地化 AI 出图工作台，支持：

- 图片批量生成
- 图片批量生成
- 套图模式与亚马逊 A+ 图模式
- 提示词模式与参考图复刻
- 历史记录、结果审核与通过项导出
- 飞书多维表格同步

## 3. 安装与启动

### 3.1 推荐安装方式

请从 GitHub Releases 下载最新 Windows 安装器。

当前对外交付安装器命名为：

```text
IMAGE-STUDIO-WINDOWS-0.4.0.exe
```

### 3.2 安装后启动

安装完成后，按安装器引导启动程序即可。

如果你拿到的是便携目录版本，请双击：

```text
启动正式版.bat
```

首次启动后，浏览器默认访问：

```text
http://127.0.0.1:3000
```

## 4. 首次使用

建议按下面顺序完成首次配置：

1. 打开“设置”页
2. 配置 Gemini 官方接口或兼容中转
3. 点击测试连接
4. 保存设置
5. 如需团队协作，再配置飞书多维表格同步
6. 返回创作台提交第一组测试任务

## 5. Gemini / 中转配置

设置页支持两种接入方式：

### 5.1 官方 Gemini

- 在“默认 API Key”中填写 Gemini API Key
- `Gemini Base URL / 中转地址` 留空
- API 版本建议保持默认

### 5.2 Gemini 兼容中转

- 在“默认 API Key”中填写中转提供的 Key
- 在 `Gemini Base URL / 中转地址` 中填写对方提供的 `base_url`
- 如果中转要求额外请求头，可填写到 `自定义请求头 JSON`

## 6. 飞书多维表格同步

如需把生成结果自动同步到飞书多维表格，需要在设置页填写：

- 飞书 App ID
- 飞书 App Secret
- 多维表格 App Token
- 多维表格 Table ID
- 字段映射 JSON

建议直接在飞书多维表格中创建以下字段：

- 标题：单行文本
- 生成图片：附件 / 图片
- 生图模式：单行文本或单选
  若配置为单选，建议统一选项为：标准模式、提示词模式、参考图复刻模式、套图模式、亚马逊A+模式
- 语言：单行文本
- 提示词翻译：单行文本
- 真实照片优化：单行文本
- 图片统计：多行文本
- 比例汇总：多行文本
- 分辨率汇总：多行文本
- 尺寸汇总：多行文本
- 生成统计：多行文本
- 任务状态：单行文本
- 提示词摘要：多行文本
- 负向提示词：多行文本
- 生成时间：日期时间
- 任务ID：单行文本

推荐字段映射 JSON：

```json
{
  "title": "标题",
  "image": "生成图片",
  "mode": "生图模式",
  "language": "语言",
  "promptTranslation": "提示词翻译",
  "promptOptimization": "真实照片优化",
  "typeSummary": "图片统计",
  "ratioSummary": "比例汇总",
  "resolutionSummary": "分辨率汇总",
  "sizeSummary": "尺寸汇总",
  "statusSummary": "生成统计",
  "status": "任务状态",
  "prompt": "提示词摘要",
  "negativePrompt": "负向提示词",
  "createdAt": "生成时间",
  "jobId": "任务ID"
}
```

如果你的旧飞书表已经在用 `platform`、`country` 两列，也可以继续保留；系统仍兼容这两个可选字段，只是新的推荐结构默认收敛为“生图模式 + 语言”。

飞书同步时，“生图模式”字段会统一写入以下口径：

- 标准模式
- 提示词模式
- 参考图复刻模式
- 套图模式
- 亚马逊A+模式

## 7. 创作模式说明

### 7.1 标准出图

适合常规图片生成，可自由选择图型、比例、分辨率和数量。

### 7.2 套图模式

适合一次生成整套图片，固定覆盖：

- 主图
- 生活方式
- 卖点总览
- 场景图
- 材质工艺
- 尺寸参数

### 7.3 亚马逊 A+ 图模式

适合生成一套亚马逊 A+ 图片详情图，固定覆盖：

- 主视觉海报
- 核心卖点解析
- 多场景应用展示
- 产品细节
- 尺寸标注
- 文化价值

### 7.4 提示词模式

适合直接按 prompt 出图，支持两个独立开关：

- 翻译为输出语言：开启后先翻译到目标语言，再提交给模型。
- 自动优化为真实照片：开启后按当前语言优化成更接近真实商业摄影的提示词。

四种组合分别对应：原文直传、仅翻译、仅优化、先翻译再优化。

### 7.5 参考图复刻

适合把“图片原图”严格生成成“参考图”的样子。参考图是母版，系统只负责把原图主体和所有同类关联元素替换进去，其他内容尽量不变。

当前规则如下：

- 参考图只使用第一张；如果你重复选择多张，系统也只会取第一张。
- 只保留 3 个生成参数：比例、分辨率、数量。
- 不再参与国家、语言、平台、模板、复刻强度、补充提示词、负向提示词、JSON 覆盖。
- 默认文案来源为“完全参照原图”。
- 如果切换到“按文案表格”，系统会保留参考图的文字版式、字体风格和区块位置，但需要改掉的文案优先按创作台里填写的图片名、品牌名、核心卖点、补充说明生成；缺失字段会回退到原图识别。
- 任务详情页中的结果类型统一显示为“复刻图”。

## 8. 日常使用流程

推荐使用顺序如下：

1. 上传主图或参考图
2. 选择创作模式
3. 填写图片信息、卖点、材质、尺寸等字段
4. 按当前模式选择参数
   标准 / 套图 / A+ / 提示词模式：继续选择国家、语言、平台、比例、分辨率和数量
   参考图复刻：只需要确认比例、分辨率、数量和文案来源
5. 提交任务
6. 在历史记录和任务详情页查看结果
7. 对结果执行入选、通过、淘汰
8. 导出通过项，或同步到飞书多维表格

## 9. 结果查看与审核

在任务详情页中可以查看：

- 原图与参考图
- 生成图与审核操作
- 请求比例与实际返回尺寸
- 实际返回大小（MB）
- 生成警告与错误
- 审核状态

历史记录页支持：

- 按图片、生图模式、语言、时间筛选
- 缩略图预览
- 点击缩略图放大查看
- 进入任务详情页进一步审核

## 10. 常见问题

### 10.1 页面能打开，但生成失败

优先检查：

- API Key 是否有效
- Base URL 是否正确
- 自定义请求头 JSON 是否合法
- 模型名是否填写正确

### 10.2 飞书同步失败

优先检查：

- 飞书应用权限
- App Token / Table ID 是否正确
- 字段映射 JSON 是否与表结构一致
- 图片是否已经生成成功

### 10.3 生成图语言不对

标准、套图、亚马逊 A+ 模式会先将输入信息翻译到目标语言后再送入模型。请确认：

- 输出语言选择正确
- 输入字段不是空值
- 提示词模式下没有手工写入与目标语言冲突的文本

参考图复刻不再依赖市场语言参数驱动 prompt；如果文案来源选择“完全参照原图”，优先保留参考图原文字，只有明显冲突时才做最小必要修改。

## 11. 交付说明

当前对外交付包默认不会带出以下本机敏感内容：

- 默认 API Key
- 自定义请求头
- 飞书 App Secret
- 本地任务记录
- 本地素材目录

因此，接收方在安装后需要自行填写自己的接口配置与飞书配置。
